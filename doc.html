用户相关——购物袋

向购物袋中添加商品

url /UserV0_1/ShoppingBag/addBag
格式 json
HTTP请求方式 post
权限 private 
需要传递的参数：
{
    "num": 1,//商品数量
    "goods_id": "abcdefg"//商品id
}

修改购物袋中商品数量

url /UserV0_1/ShoppingBag/saveBagNum
格式 json
HTTP请求方式 post
权限 private 
需要传递的参数：
{
    "num": 1,//商品数量
    "goods_id": "abcdefg"//商品id
}

从购物袋中删除一个商品

url /UserV0_1/ShoppingBag/removeBag
格式 json
HTTP请求方式 post
权限 private 
需要传递的参数：
{
    "bag_id": "购物袋的id"
}

查找用户的购物袋列表

url /UserV0_1/ShoppingBag/showList
格式 json
HTTP请求方式 post
权限 private 
直接返回用户的购物袋参数

用户相关——订单管理

使用要求

1、先调用 订单添加接口添加订单

2、如果用户选择立刻支付，在客户端支付完成后，就可以调用 支付接口来更改订单状态为已支付待发货

添加订单

url /UserV0_1/Order/add
格式 json
HTTP请求方式 post
权限 private 
需要传递的参数：
{
    "token": "0656ae5d3ef7202248a030194008e0be",//敏感操作，必须传递令牌
    "num": "1",//用户选择的商品数量
    "goods_id": "abcdefg",//用户购买的商品id
}

支付接口

url /UserV0_1/Order/payment
格式 json
HTTP请求方式 post
权限 private 
需要传递的参数：
{
    "token": "0656ae5d3ef7202248a030194008e0be",//敏感操作，必须传递令牌
    "num": "1",//用户选择的商品数量
    "goods_id": "abcdefg",//用户购买的商品id
}

取消订单

url /UserV0_1/Order/remove
格式 json
HTTP请求方式 post
权限 private 
只有当订单未支付的时候才能取消，后台会判断，注意返回信息。                                
需要传递的参数：
{
    "token": "0656ae5d3ef7202248a030194008e0be",//敏感操作，必须传递令牌
    "order_id": "abcdefg",//订单号
}

获取单个订单

url /UserV0_1/Order/getOne
格式 json
HTTP请求方式 get
权限 private 
需要传递的参数：
{
    "token": "0656ae5d3ef7202248a030194008e0be",//敏感操作，必须传递令牌
    "order_id": "abcdefg",//订单号
}
返回此订单的数据

获得订单列表

url /UserV0_1/Order/showList
格式 json
HTTP请求方式 get
权限 private 
                    
需要传递的参数：
{
    "token": "0656ae5d3ef7202248a030194008e0be",//敏感操作，必须传递令牌
}
返回此用户所有的订单信息
